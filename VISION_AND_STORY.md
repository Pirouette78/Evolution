# VISION & USER JOURNEY: MACRO-EVOLUTION

## 1. L'Expérience de Jeu (The "Feel")
Le jeu doit être une œuvre d'art contemplative autant qu'un jeu de stratégie. 
Le joueur ne contrôle pas directement les unités, mais modifie les règles de leur vie. 
L'esthétique est organique (Slime, Fluides, Nuées).

## 2. Le Cycle de Progression (The Loop)
1. **Infection :** Le joueur commence avec une cellule dans un environnement (ex: Humain).
2. **Consommation :** Les entités mangent pour gagner de l'énergie.
3. **Évolution :** Le joueur dépense l'énergie dans un arbre technologique (JSON).
4. **Expansion :** Les entités se divisent et s'étendent.
5. **Domination :** À 100% de contrôle, le joueur est "promu" à l'échelle supérieure.

## 3. Les Phases de l'Histoire
- **Phase 1 (Cellule) :** Gameplay microscopique dans un corps humain (inconnu au début).
- **Phase 2 (Tactique Slime) :** Vue macro de l'infection. Le corps humain devient une carte de chaleur (Heatmap).
- **Phase 3 (Zombies) :** L'humain infecté sort dans le monde. On gère des nuées de zombies sur une carte de ville/planète.
- **Phase 4 (Espace) :** Les zombies lancent des fusées. La Terre devient une entité parmi d'autres dans le système solaire.
- **Final :** Domination galactique et universelle.

## 4. Architecture Multijoueur "RGB"
- Chaque instance de simulation (un humain, une planète) accueille **3 joueurs maximum**.
- Joueur 1 = Rouge, Joueur 2 = Vert, Joueur 3 = Bleu.
- Les zones de conflit créent des couleurs secondaires (Jaune, Cyan, Magenta).
- Les perdants d'une phase deviennent spectateurs et peuvent observer toutes les instances actives dans l'univers.
